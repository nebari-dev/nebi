# Tutorial

This tutorial walks you through Nebi from first use to team collaboration. Each page builds on the previous one, so it's best to read them in order.

**What you'll need:**

- [Pixi](https://pixi.sh) installed and on your PATH
- [Nebi](../getting-started.md) installed
- Basic familiarity with Pixi (creating projects, adding dependencies)

If you're new to Pixi, start with the [Pixi documentation](https://pixi.sh/latest/) first. Nebi manages Pixi workspaces, so you'll want to be comfortable with `pixi.toml`, environments, and tasks before diving in.

## What You'll Learn

1. [**First Steps**](./first-steps.md) — Initialize and track your first workspace
2. [**Working with Workspaces**](./working-with-workspaces.md) — List, activate, and manage workspaces by name
3. [**Sharing with a Server**](./sharing-with-a-server.md) — Connect to a Nebi server and push/pull environments
4. [**Versions and Tags**](./versions-and-tags.md) — Understand content-addressed versioning and tagging
5. [**Status and Diffing**](./status-and-diffing.md) — Track changes and compare environments
6. [**Publishing to OCI**](./publishing-to-oci.md) — Distribute environments through OCI registries
7. [**Working as a Team**](./working-as-a-team.md) — Multi-person workflows and best practices
